There are some problems with using “bags of properties.”

Code reviewers may have to work harder to determine:

  • when a bag contains properties from an untrusted source
    (this can lead to careless mixing of trusted and untrusted properties),
  • when a bag contains sensitive properties that need to be treated with care,
  • whether an Object.assign or $.extend call reads from or writes into a bag.

To mitigate:
const URL_KEY = Symbol('url') // Define a symbol key let obj = { [URL_KEY]: secret } // Symbols not stringified
