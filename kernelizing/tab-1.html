Security in Dynamic Systems

  • We are only running trustworthy code because only sources written or vetted by trusted developers make it into our repo. These deployed bundles finish building and the process's code segment is marked read-only before untrusted inputs reach them.

  • We won't leak a password reset URL because the code that allocates them encapsulates them in a ResetLink object. The URL is only available via the getURL method which is only called by the email sender which has been carefully reviewed with leaks in mind.

  • We can put confidential user data in origin https://mycompany.com because we only serve HTML that comes from static source files or that was generated by carefully reviewed template systems which contextually filter all inputs.
